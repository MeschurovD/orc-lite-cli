## Открытые вопросы

1. **Повторяющиеся cron-задачи**: после выполнения задачи с cron-расписанием, статус должен сбрасываться обратно в `pending`? Или использовать отдельный статус `recurring`?
   > Предложение: для cron-задач не менять статус на done, а вести лог выполнений отдельно. Статус `pending` сохраняется для следующего запуска.

2. **Daemon + run конфликт**: что если daemon запущен и пользователь вызывает `orc-lite run`? Нужен lock-файл?
   > Предложение: да, lock-файл `.orc-lite/run.lock`. `run` и `daemon` проверяют его перед стартом.

3. **Таймзона**: cron-расписание в какой таймзоне? Локальная системная?
   > Предложение: локальная по умолчанию, опция `timezone` в конфиге для явного указания.

4. **Просроченные задачи**: если daemon запущен и время задачи уже прошло - запустить немедленно или пропустить?
   > Предложение: запустить немедленно если прошло не более 1 часа, иначе пропустить с предупреждением.
